Segment Analysis

GPS Deviation

Perpendicular distance from each route to the reference route (marked REF — the first selected route with recorded time, if any). "Show" colors that route's map line green → red, from on-course up to its own worst 5% of points — the scale is per-route, not a fixed distance.

Distance Drift

Cumulative distance difference vs. the reference route (the first selected route, marked REF), sampled at equal elapsed time. A steadily diverging value means a device reads consistently long or short.

Running Dynamics

Average vertical oscillation, ground contact time, vertical ratio, GCT balance, and step length per route — FIT-only, needs a footpod or the watch's own running-dynamics accelerometer feature enabled. Coverage shows what fraction of points had this data.

Session Self-Check

Compares each device's own self-reported distance/time/ascent/descent (from the FIT session summary) against what's recomputed from its raw track. A mismatch is a firmware self-reporting bug by definition. When the file includes raw barometric pressure, also compares the device's reported elevation gain against an independently derived barometric estimate.

HR / Cadence Validation

Compares each route's heart rate against the reference route's (marked REF — typically a chest strap): mean absolute error and bias. Also flags stretches where a route's HR tracks its own cadence instead of pulse — a classic optical heart-rate failure mode.

Dropout Diagnostics

Recording gaps (time between consecutive points beyond what the file's own sampling rate would predict) and null-value runs per sensor metric, per route.
